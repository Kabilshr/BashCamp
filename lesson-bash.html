What is Bash?

A quick primer on the Bash shell — what it is, why it’s used everywhere, and how it powers your command line and automation.

Start Reading Take Quiz

Bash in one minute

  • Bash = Bourne Again SHell — a command interpreter (shell) and scripting language.
  • Default shell on most Linux distros and available on macOS & Windows (via WSL, Git Bash, MSYS2).
  • Great for: running commands, chaining tools with pipes, and automating routine tasks with scripts.
  • You use it in a terminal: type a command, press Enter, see output.
# where am I?
$ pwd
/home/user/projects

# what’s here?
$ ls -la

# run a command and save output
$ date >> logs/run.log

Why do developers use Bash?

Bash is lightweight, ubiquitous, and works hand‑in‑hand with the Unix philosophy: combine small tools (grep, sed, awk, sort, wc) to solve bigger problems. It’s ideal for development workflows, servers, CI/CD, and quick data wrangling.

Strengths

  • Everywhere: Linux, macOS, WSL
  • Fast for ad‑hoc tasks & automation
  • Powerful piping & redirection
  • Works with any CLI tool

Considerations

  • Not ideal for large apps
  • Quoting/escaping can be tricky
  • Portability requires care

Starting Bash

Open a terminal and you’re likely already in Bash. If not, you can start it explicitly: 

$ bash

The prompt (often ending with $) is where you type commands. Use Ctrl+C to cancel a running command, ↑/↓ to browse history, and Tab for auto‑completion.

First commands

# current directory
$ pwd

# list files (long, all)
$ ls -la

# change directory
$ cd /etc
$ cd -   # back
# make a folder and file
$ mkdir demo
$ echo "hello" > demo/hello.txt

# view file
$ cat demo/hello.txt

Pipes & redirection

Use a pipe (|) to send the output of one command to another. Redirect with > (write) or >> (append). 

# count 404 lines in access.log
$ cat access.log | grep 404 | wc -l

# append the current date to a file
$ date >> logs/run.log

Variables & quoting

Bash variables are simple key=value pairs (no spaces):

$ NAME="Bash"
$ echo "Hello, $NAME"
Hello, Bash
  • "double quotes" expand variables; 'single quotes' do not.
  • Export a variable to make it available to child processes: export NAME.

Writing your first script

A Bash script is a text file with commands. Start with a shebang to tell the system how to run it: 

$ nano hello.sh
#!/usr/bin/env bash
echo "Hello from a script!"
# make it executable and run
$ chmod +x hello.sh
$ ./hello.sh

When to use Bash vs GUI

  • Use Bash for repeatable tasks, quick data processing, server admin, CI jobs, and glue code.
  • Use a GUI for visual work (design, spreadsheets) or when discoverability matters.
Tip: if you repeat something more than twice, consider making a small Bash script.
